01 — Project Overview

One platform.
Every campus need.

Study Mart replaces fragmented WhatsApp groups, notice boards, and generic marketplaces with a single verified, campus-scoped platform. Every user is authenticated via their institutional email — so every interaction happens within a trusted community.

05 — Tech Stack

Chosen for speed.
Built to scale.

Every technology was selected with a clear purpose — no bloat, no trend-chasing.

⚡ Next.js v14

App Router with SSR/SSG support, optimized image handling, and built-in middleware for protected routes.

🎨 Tailwind CSS v3

Utility-first CSS with zero runtime overhead. Consistent design system enforced at the class level.

🔴 Redux Toolkit RTK

RTK Query handles all API calls with caching and refetching. Custom WebSocket middleware for chat state.

🐍 FastAPI 0.110

Async Python framework with auto-generated OpenAPI docs, native WebSocket support, and Pydantic validation.

🍃 MongoDB 7.x

Flexible document model suits heterogeneous schemas across listings, events, and messages. Motor for async I/O.

☁️ Cloudinary CDN

CDN-delivered media hosting with on-the-fly transformations. Signed uploads for production security.

🐳 Docker Compose

Full environment parity from local dev to production. Production compose adds Nginx and Gunicorn workers.

📱 React Native Planned

Same Redux store shape as the web app. Backend is already fully mobile-ready with no API changes required.

06 — Environment Variables

Zero-config setup.
Copy and run.

Copy .env.example to .env and fill in your values. Never commit your .env file.

.env.example
# Application
APP_ENV=development # development | staging | production
APP_PORT=8000
FRONTEND_URL=http://localhost:3000
# MongoDB
MONGO_URI=mongodb://localhost:27017/studymart
MONGO_DB_NAME=studymart
# JWT Auth
JWT_SECRET_KEY=your-256-bit-secret-key-here
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=7
# Cloudinary
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
CLOUDINARY_UPLOAD_PRESET=studymart_unsigned
# Campus Verification
ALLOWED_EMAIL_DOMAINS=university.edu,college.ac.in
# Next.js Public Variables
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000/api/v1
NEXT_PUBLIC_WS_URL=ws://localhost:8000/ws
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your_cloud_name
NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET=studymart_unsigned
07 — Local Development Setup

Running in
under 5 minutes.

1
Clone the repository
BASH
git clone https://github.com/your-org/study-mart.git
cd study-mart
2
Backend setup (FastAPI)
BASH
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
# API docs → http://localhost:8000/docs
3
Web frontend setup (Next.js)
BASH
cd apps/web
npm install
cp ../../.env.example .env.local  # Fill in NEXT_PUBLIC_* values
npm run dev
# Web app → http://localhost:3000
4
Or use Docker (recommended)
BASH
docker compose -f docker/docker-compose.yml up --build
# backend  → http://localhost:8000
# web      → http://localhost:3000
# mongo    → localhost:27017

12 — Security

Campus-verified.
Zero-trust internals.

🔑

JWT Auth Flow

Short-lived access tokens (15 min) + httpOnly refresh tokens (7 days). Silent refresh via /auth/refresh. No tokens in localStorage.

🔒

bcrypt Passwords

Passwords hashed with bcrypt at cost factor 12. Verification uses constant-time comparison to prevent timing attacks.

🏫

Campus Isolation

Every query is scoped to the authenticated user's campus field. Cross-campus data access is architecturally impossible.

✅

Email Verification

Registration requires OTP confirmation of a campus email domain. Domains are whitelisted via ALLOWED_EMAIL_DOMAINS.

⚡

Rate Limiting

slowapi middleware applied to all auth endpoints at 10 requests/min per IP.

🛡️

Pydantic Validation

All request bodies validated with Pydantic models before any DB operation. No raw user input reaches MongoDB queries.
